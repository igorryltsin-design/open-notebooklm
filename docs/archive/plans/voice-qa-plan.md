# План: Голосовой Q&A по документу

## Цель MVP
- [ ] Пользователь задаёт вопрос голосом по текущему документу
- [ ] Система распознаёт вопрос (STT)
- [ ] Система отвечает по документу (RAG + LLM)
- [ ] Ответ озвучивается (TTS)

## Фаза A (MVP): push-to-talk
- [x] Кнопка записи вопроса в UI (`Начать/Остановить`)
- [x] Отправка аудио на backend (`multipart/form-data`)
- [x] Backend endpoint `POST /api/voice_qa/{document_id}`
- [x] STT распознавание аудио -> текст вопроса
- [x] Ответ по документу через RAG + LLM
- [x] Возврат `question_text`, `answer_text`, `sources`
- [x] Озвучка ответа (TTS) и воспроизведение в UI
- [x] UI статус этапов: `Слушаю / Распознаю / Думаю / Говорю`

## Фаза B: диалоговый режим
- [x] История голосовых Q/A в UI
- [x] Ограниченный контекст истории (последние N сообщений)
- [x] Режим `Строго по источникам`
- [x] Кнопка `Остановить озвучку`

## Фаза C: UX и качество
- [x] Подсветка/показ использованных источников (чанков)
- [x] Настройка длины ответа (`short / medium / long`)
- [x] Таймауты и понятные ошибки (`STT`, `LLM`, `TTS`)
- [x] Retry на временных ошибках (по необходимости)

## Фаза D (опционально): near-realtime
- [x] VAD (автоопределение конца речи)
- [x] Streaming STT (частичные транскрипты)
- [x] Streaming LLM/TTS
- [x] `barge-in` (перебить ответ новым вопросом)
