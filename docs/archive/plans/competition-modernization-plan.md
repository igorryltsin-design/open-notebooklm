# Конкурсный план развития Open NotebookLM

## Цель
Собрать конкурсный релиз, который демонстрирует не только генерацию ответов, но и полноценную исследовательскую студию: загрузка источников, проверяемый RAG, генерация скриптов, аудио-пайплайн и сильный UX.

## Фаза 1. Базовая стабильность (1 неделя)
- Зафиксировать UX-ошибки текущего интерфейса и закрыть критичные баги.
- Выровнять состояние между Sources, Chat, Studio (сбросы, переключение документов, batch jobs).
- Довести фронтенд до стабильной сборки + smoke-тесты основных сценариев.
- Результат: стабильный `v1` без блокирующих регрессий.

## Фаза 2. Современный UI/UX (1 неделя)
- Единая дизайн-система (токены цветов, типографика, радиусы, тени, анимации).
- Редизайн shell-уровня: topbar, секции, статусные карточки, модальные окна.
- Улучшение UX плотных сценариев: загрузка источников, навигация, чат, генерация.
- Адаптивность для desktop/laptop/tablet без потери функций.
- Результат: визуально цельный, «конкурсный» интерфейс.

## Фаза 3. Функциональный апгрейд (2 недели)
- Project Notebook как центральная сущность для сценариев исследования.
- Улучшение Conv RAG и режима сравнения документов.
- Улучшение пайплайна генерации подкаст-скриптов (контроль сценария и структуры).
- Улучшение управления ролями/моделями/голосами для воспроизводимых результатов.
- Результат: ощутимый прирост практической ценности продукта.

## Фаза 4. Надежность и качество (1 неделя)
- Набор интеграционных тестов по API и UI-критическим маршрутам.
- Проверка производительности на больших наборах документов.
- Наблюдаемость: понятные статусы, ошибки и диагностические сообщения.
- Результат: предсказуемая работа на демо-нагрузке.

## Фаза 5. Конкурсная упаковка (1 неделя)
- Подготовка демонстрационного сценария «от загрузки до аудио результата».
- Скриншоты/видео и презентация с фокусом на измеримую пользу.
- Финальная стабилизация и freeze ветки релиза.
- Результат: готовый набор для защиты проекта.

## Метрики готовности
- UI: единый визуальный стиль во всех ключевых панелях.
- UX: полный сценарий выполняется без ручных «обходов».
- Stability: сборка и тесты проходят стабильно.
- Demo-readiness: демонстрация основного сценария < 5 минут.
